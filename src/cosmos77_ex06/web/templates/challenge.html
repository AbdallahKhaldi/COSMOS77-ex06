01 · THE GAME

A pursuit two autonomous agents play in plain English

This is a top-down Cops & Robbers chase formalized as a Dec-POMDP: a decentralized, partially-observable game. The cop hunts; the thief evades. Neither agent sees the whole board — each gets only a vision-radius window around itself, so both must reason and bluff under uncertainty.

Each side is a FastMCP server that exposes tools only — look, move, place_barrier, say — and holds no intelligence of its own. The LLM lives exclusively in the orchestrator (the MCP client). The orchestrator pulls each agent's observation, asks the model for a move plus one line of trash talk, and pushes it back over the wire. Agents coordinate and taunt in free natural language, never a rigid numeric protocol — a coordinate-leak guard flags any message that smells like raw grid math.

03 · THE SHARED TOKEN

One bearer token, agreed by both sides

Every tool call into either server carries a bearer token. A challenge only runs if both teams agree on the same token at coordination time — it's the handshake that proves the match was set up by humans on both ends, not a drive-by.

  1. Agree out-of-band. Pick one shared secret together over chat/email — any high-entropy string (e.g. openssl rand -hex 16). Whoever runs the orchestrator must use the exact same value the other side loaded into its servers.
  2. Keep it out of the repo. Each side stores it in .env as BONUS_MCP_TOKEN=… (gitignored, per our Rule 9). It never gets committed; rotating it instantly revokes old access.
  3. Hand it to the run. In the console, paste it into the shared clearance token field. On the CLI it's read from your environment / the partner config. If the token mismatches, calls are rejected and the game won't start.

Output prints totals_by_group and the bonus_claim, and writes reports/bonus_game.json — the file both sides will diff and email.

06 · THE INTER-GROUP BONUS (E12)

10 points on the table — if the JSON matches to the byte

The headline prize. Two groups play a 6-game role-swap series: three sub-games with our cop vs your thief, three with your cop vs our thief, so neither side is advantaged by a role. A single orchestrator engine runs all six.

  1. Play the series. Run the bonus command above. Each side's points across its three cop and three thief sub-games are summed into totals_by_group.
  2. Serialize canonically. Both codebases write the bonus_game report through the same canonical serializer — sort_keys=True, ensure_ascii=False, indent=2. Same agreed value object in, identical bytes out, regardless of key insertion order.
  3. Diff-check together. Before anyone hits send, both sides run the byte-for-byte diff over the two files. It's deliberately dumb-and-total — it compares raw bytes and, on a mismatch, points at the first differing offset and the offending field. 
    uv run python -c "from cosmos77_ex06.bonus.diff_check import diff_files, format_result as f; print(f(diff_files('reports/bonus_game.json', 'partner_bonus_game.json')))"
IDENTICAL (N bytes) -> safe to set mutual_agreement:true and send
  4. Email the professor. Once the diff is clean, set mutual_agreement: true and each group separately emails the identical bonus_game JSON to rmisegal+uoh26b@gmail.com before the Friday 08:30 deadline.
⚠ Mismatch = zero for both. If the two files differ by a single byte — a float score, a reordered list, a flipped orientation, a missing sort_keys — neither group scores. The diff-check is the precondition for sending; do not skip it.
